# OctoAcme RACI-Lite Template

Use this template to map role ownership for a specific project or workstream. Fill in the roles relevant to your team and mark each cell with **R**, **A**, **C**, or **I** (or leave blank if not applicable).

**Key:**
- **R** — Responsible: does the work
- **A** — Accountable: owns the outcome; approves the deliverable (one per row)
- **C** — Consulted: provides input before or during the activity
- **I** — Informed: kept up to date but not directly involved

> For full role definitions and a pre-filled example, see [Roles & Personas](octoacme-roles-and-personas.md).

## Usage Notes

1. **One Accountable (A) per row** — if you find multiple As, clarify ownership before the phase begins.
2. **Limit Consulted (C)** — consult only those whose input meaningfully shapes the outcome; over-consulting slows decisions.
3. **Keep Informed (I) lean** — include stakeholders who need awareness but do not act on the information day-to-day.
4. **Review at kickoff** — walk through the RACI with the team during project initiation to surface gaps and conflicts early.
5. **Revisit at phase transitions** — re-validate ownership when moving from Planning → Execution or Execution → Release.

---

## Checklist: Before You Start a Phase

Use this checklist at each phase transition to confirm ownership is clear.

- [ ] Each activity in the phase has exactly one Accountable (A) role assigned
- [ ] All Responsible (R) roles are aware of and have accepted their tasks
- [ ] Consulted (C) parties have been briefed and know when their input is needed
- [ ] Informed (I) parties are included in the relevant status updates or channels
- [ ] Blockers or gaps in coverage are logged and escalated to the Project Manager
- [ ] The RACI has been reviewed by the Project Manager and Product Manager
